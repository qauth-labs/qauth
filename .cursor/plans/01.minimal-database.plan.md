# P0 Minimal Database Setup

## Overview

Set up the foundational database infrastructure for Phase 0 - configuration only, no actual table schemas.

## Implementation Steps

### 0. Open Issue & Create Branch

- Open detailed issue in GitHub
- Checkout new branch to start (<type>/<issue-number>-<issue-slug>)

### 1. Environment Configuration

- Update `.env.example` with comprehensive database configuration template
- Add comments explaining each environment variable
- Include example PostgreSQL connection string format

### 2. Database Connection Module

Create `libs/data-access/db/src/lib/db.ts`:

- Export database connection instance using `drizzle()`
- Import from `drizzle-orm/node-postgres`
- Use `pg` Pool for connection management
- Export both `db` instance and `pool` for direct access if needed

### 3. Fix Drizzle Config Path

Update `libs/data-access/db/drizzle.config.ts`:

- Fix schema path from `./src/schemas/index.ts` to `./src/lib/schema/index.ts`
- Update migrations output path to `./drizzle` (standard location)
- Ensure dotenv path is correct

### 4. Update Package Exports

Update `libs/data-access/db/src/index.ts`:

- Export database connection and pool
- Export schema (even though empty for now)
- Add TypeScript types export for future use

### 5. Add Migration Scripts

Update `libs/data-access/db/project.json`:

- Add `db:generate` target for generating migrations
- Add `db:migrate` target for running migrations
- Add `db:studio` target for Drizzle Studio
- Use Nx executor for consistent tooling

### 6. Documentation

Update `libs/data-access/db/README.md`:

- Document setup process
- Add usage examples for database connection
- List available Nx commands
- Note that actual schemas will be implemented in P1

## Files to Modify

- `.env.example` - Add database environment variables
- `libs/data-access/db/drizzle.config.ts` - Fix paths
- `libs/data-access/db/src/lib/db.ts` - Create connection module
- `libs/data-access/db/src/index.ts` - Export connection
- `libs/data-access/db/project.json` - Add migration targets
- `libs/data-access/db/README.md` - Documentation

## What This Achieves (P0 Scope)

- PostgreSQL connection configuration ready
- Drizzle ORM installed and configured
- Migration system ready to use
- Database connection module exportable
- Foundation ready for P1 schema implementation

## What's Deferred to P1

- Actual table schemas (users, sessions, oauth_clients, etc.)
- Database migrations with real tables
- CRUD operations
