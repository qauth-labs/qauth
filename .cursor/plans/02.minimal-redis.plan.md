# P0 Redis Connection Setup

## Implementation Steps

### 0. Create GitHub Issue

- Create GitHub issue with MCP in qauth-labs/qauth
- Checkout new branch
-

### 1. Create Redis Library Structure

Generate new library using Nx:

- Create `libs/data-access/cache` library for Redis
- Configure as buildable library with TypeScript
- Add appropriate tags: `scope:data-access`, `type:cache`

### 2. Install Redis Dependencies

Add required packages:

```bash
pnpm add ioredis
pnpm add -D @types/ioredis
```

**Why ioredis?**

- Better TypeScript support than node-redis
- Built-in cluster support
- Promise-based API
- Better connection handling
- Used by major projects (NestJS, Bull)

### 3. Update Environment Configuration

Update `.env.example`:

- Add `REDIS_URL` with connection string format
- Add `REDIS_HOST`, `REDIS_PORT` for alternative config
- Add `REDIS_PASSWORD` for authentication
- Add `REDIS_DB` for database selection
- Add connection pool settings (max connections, timeout, etc.)
- Add comprehensive comments

### 4. Create Redis Connection Module

Create `libs/data-access/cache/src/lib/redis.ts`:

- Export Redis connection instance using `ioredis`
- Configure connection options from environment variables
- Add connection error handling and retry logic
- Export utility functions: `testConnection()`, `closeRedis()`
- Add graceful shutdown support
- Configure clustering support (optional, for future)

Key features:

```typescript
- Redis client with retry strategy
- Connection health check
- Graceful shutdown
- Error event handlers
```

### 5. Create Redis Utilities

Create `libs/data-access/cache/src/lib/utils.ts`:

- Session storage helpers (set, get, delete with TTL)
- Rate limiting utilities (check and increment counters)
- Cache wrapper functions (get/set with JSON serialization)
- Key naming conventions (prefixes for different data types)

Utility examples:

```typescript
-setSession(sessionId, data, ttl) -
  getSession(sessionId) -
  deleteSession(sessionId) -
  checkRateLimit(key, limit, window);
```

### 6. Update Package Exports

Update `libs/data-access/cache/src/index.ts`:

- Export Redis client instance
- Export all utility functions
- Export TypeScript types
- Export constants (key prefixes, default TTLs)

### 7. Add Nx Project Configuration

Create/update `libs/data-access/cache/project.json`:

- Add project metadata
- Configure build target if needed
- Add lint and test targets
- Set proper tags

### 8. Documentation

Create `libs/data-access/cache/README.md`:

## Key Design Decisions

1. **ioredis over node-redis**: Better TypeScript support and features
2. **Utility layer**: Abstract common patterns for consistency
3. **Key prefixes**: Namespace different data types (`session:`, `rate:`, etc.)
4. **TTL defaults**: Sensible defaults, configurable per use case
5. **Error handling**: Graceful degradation where possible
